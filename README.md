# Low-Code Platforms Database

<h3>Постановка задачи</h3>
Разработано консольное приложение на языке C для управления файловой базой данных low-code платформ. Программа обеспечивает полный цикл работы с данными: создание, хранение, редактирование, поиск и сортировку записей о low-code платформах с различными техническими и бизнес-характеристиками. Данные сохраняются в бинарном файле для обеспечения долговременного хранения и быстрого доступа.

<h3>Структура проекта</h3>
Проект реализован в виде монолитного консольного приложения с модульной структурой. Основной файл main.c содержит всю логику программы, организованную в следующие логические блоки:

- Структуры данных - определения типов LowCodePlatform и PlatformList для хранения информации о платформах и управления динамическим массивом записей

- Управление памятью - функции для работы с динамическим массивом: инициализация, расширение, добавление элементов и освобождение ресурсов

- Ввод-вывод - безопасные функции для чтения различных типов данных с консоли с валидацией

- Файловые операции - работа с бинарным файлом для сохранения и загрузки всей базы данных, а также прямой доступ к отдельным записям

- Бизнес-логика - алгоритмы поиска по диапазону значений, сортировки и генерации тестовых данных

- Пользовательский интерфейс - текстовое меню с навигацией и функциями отображения данных

<h3>Алгоритм работы программы</h3>

- Инициализация динамического массива и загрузка данных из файла (если существует)

- Отображение главного меню с доступными операциями

- Обработка выбора пользователя с вызовом соответствующих функций

- Циклическое выполнение до команды выхода

- Автоматическое сохранение данных при завершении работы

<h3>Описание ключевых функций</h3>
<h4>Управление данными</h4>

- init_list() - создает динамический массив с начальной емкостью 16 элементов

- push_back() - добавляет новую запись в конец массива, автоматически расширяя его при необходимости

- ensure_capacity() - удваивает емкость массива при достижении предела
  
- free_list() - корректно освобождает всю выделенную память

<h4>Ввод данных с валидацией</h4>

- read_int_in_range() - читает целое число, проверяя его нахождение в заданных границах

- read_double_nonneg() - читает вещественное число, гарантируя неотрицательное значение

- read_bool() - обрабатывает логический ввод (0/1) с защитой от некорректных данных

- read_line() - безопасно читает строку, удаляя символ новой строки
  
<h4>Файловые операции</h4>

- save_all() - сохраняет весь массив записей в бинарный файл одной операцией

- load_all() - загружает все записи из файла, определяя их количество по размеру файла

- read_record_by_index() - обеспечивает прямой доступ к любой записи по индексу без загрузки всего файла

<h4>Бизнес-функции</h4>

- search_by_range() - выполняет поиск записей, у которых значение выбранного поля находится в заданном диапазоне

- generate_random_records() - создает тестовые данные со случайными значениями для отладки

- print_platform() - форматирует вывод информации о платформе в читаемом виде

- sort_by_normalized_dev_hours() - заглушка для реализации алгоритма сортировки

<h4>Пользовательский интерфейс</h4>

- menu() - отображает главное меню программы

- print_title_page() - показывает титульную страницу с информацией о проекте

- Интерактивное управление через консоль с обработкой всех возможных ошибок ввода

<h3>Формат данных</h3>
<h4>Структура записи в памяти</h4>

Каждая запись о low-code платформе хранится в структуре LowCodePlatform, которая содержит:

- name[64] - название платформы (максимум 63 символа + нуль-терминатор)

- dev_hours - время разработки в часах (вещественное число, 8 байт)

- performance - оценка производительности от 0.0 до 10.0 (вещественное число, 8 байт)

- cost - стоимость лицензии (вещественное число, 8 байт)

- integrations - количество доступных интеграций (целое число, 4 байта)

- vis_level - уровень визуализации (enum: 0=Basic, 1=Medium, 2=Advanced, 4 байта)

- customization - уровень кастомизации (enum: 0=Low, 1=Medium, 2=High, 4 байта)

- mobile_support - поддержка мобильных устройств (bool, 1 байт)

<h4>Общий размер структуры: ~101-108 байт (с учетом выравнивания в памяти)</h4>
